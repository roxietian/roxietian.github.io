Line 3 · New Capability Modules on the Reporting Platform
new PO vacancy scope · cross-team enabler · innovation-lab lead dev (AI) · data-layer dev · DevOps

A · AI Assistant

select text → secure in-editor sidebar → intent recognition → RAG retrieval: client knowledge base + regulatory texts (ESRS · GRI · ISSB) → suggestion preview → user approves → audit trail entry

shipped / shipping
  • shorten · rewrite · table ↔ paragraphs · formatting fixes
roadmap
  • chapter-level rewrite · suggest-and-replace UX

  • assisted XBRL tagging: concept suggestions · extension anchoring · block tagging

B · Data Integration Layer

today: ~200 source systems → data warehouse → Excel last mile

target: modern finance systems → REST / JSON APIs → disclosures calculated in-platform

  • financial statements · remuneration data first candidates
  • API contracts + acceptance criteria = PO's daily work
  • B2B, third-party enabler focus · little end-user surface

C · Cloud Development

today: private cloud + AWS + GCP in parallel → target: simplified setup + provisioning

  • infra-focused · internal-facing
  • DevOps engineer as key partner
Design philosophy across all AI work: assisted, not automated — suggested by AI, approved by the user, every action auditable. The auditor must never hear "AI came up with that."
market scan capability race rather than product race — benchmark: Workiva AI features · gen-AI document tooling entering reporting space
