16 · STAR — AI Hotline

Hook → S/T → Action×4 → Result → Aha → Lesson
Hook
The hardest part wasn't building the AI → my own local teams were quietly refusing to let it go live → and honestly they had a point
S/T
I led the hotline across 7 markets → teams manually screened L1, with 1,000+/wk total inbound → the problem was speed AND scalability
Act 1
So I sat with the teams → two real concerns surfaced: extra validation work, and who's liable if AI is wrong
Act 2
First I clarified the use-case boundary → L1 & knowledge only, never repair, liability, SLA or closure
Act 3
Then I mapped the reporting flow with R&D → if it can't answer, it opens a Salesforce ticket → into the O&M workflow
Act 4
When validation stalled, I started 3 pilot markets → reframed it as cutting their own L1 later → and escalated for resources
Result
Once 3 pilots proved value, all 7 joined → 100+ clients, ~50/day peak, ~50% automated, plus 24/7 coverage
Aha
My aha → the hard part was designing trust, not building AI
Lesson
I learned → start with why people resist and make the value real → AI adoption is a trust problem, solved by boundaries & accountability, not a better model

17 · STAR — 0-1 Service Ecosystem

Hook → S/T → Aha → Action → Result → Lesson
Hook
The twist → my first client almost broke my own design → I built a CRM service ecosystem from scratch, no benchmark → we shipped Shell-specific to hit a deadline → then change requests flooded in
S/T
It was 5 connected platforms → every client had different SLA, packages, permissions & reporting → my job was to scale to hundreds, not just Shell
Aha
When I dug in → the requests weren't noise → everything was hardcoded to one client → the product model itself wouldn't scale
Action
So I rebuilt it → one standard core + a configurable rule layer → same workflow, only the rules differ → engineering pushed back, so I used evidence not vision → and rolled it out iteratively
Result
Now onboarding became configuration vs new development → 200+ clients, 3,000+ users → it killed repeated engineering & one-offs → and became the benchmark
Lesson
The lesson → feedback works on two levels → I now ask "different process, or same process with different rules?" → a rule you configure, a process you build

18 · STAR — Training Platform

Hook → S/T → Action×3 → Result → Aha → Lesson

S/T
