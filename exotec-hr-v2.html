16 · STAR — AI Hotline

Hook → S/T → Act×4 → Result → Aha → Lesson
Hook
the hardest part wasn't building the AI → my own local teams were quietly refusing to go live → and honestly, they had a point
S/T
I led the rollout · 7 markets · 1,000+ requests/wk → not only speed → scalability
Act 1
I sat down with the teams → two concerns: extra work · who's responsible if AI is wrong
Act 2
first, I clarified the use-case boundary → L1 + knowledge only · never repair/liability/SLA/closure
Act 3
second, I mapped the reporting workflow with R&D → Salesforce ticket → O&M workflow
Act 4
third, validation slow → started with three pilot markets → reframed the value · escalated for resources
Result
3 pilots proved it → all 7 markets · 100+ clients · ~50/day peak · ~50% automated · plus 24/7
Aha

Lesson
first, start with why people resist · make value real, not push harder → second, adoption is a trust problem → boundaries · accountability, not a better model

17 · STAR — 0-1 Service Ecosystem

Hook → S/T → Aha → Action → Result → Lesson
Hook
a 0→1 story, but the twist → my first client almost broke my own design → ecosystem from scratch · no benchmark → Shell first → shipped Shell-specific → change requests flooded
S/T
it was 5 connected platforms → every client diff SLA · packages · permissions · reporting → my job → scale to hundreds
Aha
looked like first-release noise BUT the pattern → hardcoded to one client → the product model wouldn't scale
Action
so I changed direction → one standard core + configurable rule layer → only the rules differ → engineering pushed back, so I used evidence, not a vision → iterative, in phases
Result
onboarding became configuration vs new development → 200+ clients · 3,000+ users → this model became the benchmark
Lesson
now I ask one question first → different process, or same process with different rules? → rule→configure · process→build → sometimes you build the benchmark
