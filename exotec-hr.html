17 · STAR — 0-1 Service Ecosystem

Hook → S/T → Aha → Action → Result → Lesson
Hook
first client almost broke my own design · CRM service ecosystem from scratch · EVSE · no benchmark · Shell first → shipped Shell-specific to hit deadline → change requests flooded
S/T
5 connected platforms · every client = diff SLA · packages · permissions · reporting → my job: scale to hundreds, not Shell's backlog
Aha
change requests ≠ first-release noise → all hardcoded to one client → the product model itself won't scale
Action
one standard core + configurable rule layer · same core workflow vs only the rules differ · engineering pushed back → used evidence not vision (Shell's cost · repeated RFP rules · pipeline) · made it iterative · phased
Result
onboarding = configuration vs new dev project → 200+ clients · 3,000+ users · killed repeated eng · manual setup · one-off exceptions → became the benchmark
Lesson
feedback on 2 levels: broken feature vs wrong-built product → different process, or same process diff rules? · rule→configure · process→build · 0→1: sometimes you build the benchmark
